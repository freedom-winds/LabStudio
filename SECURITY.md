# Security Policy / 安全策略

[English](#english) | [中文](#中文)

## English

### Supported Versions

Security updates are provided for the current main branch of this repository.

### Reporting a Vulnerability

If you discover a security issue, please report it privately to the project maintainer's email throught github instead of opening a public issue. Include the affected component, reproduction steps, expected impact, and any relevant logs or screenshots.

### Response Process

We will acknowledge valid reports as soon as practical, investigate the issue, and provide remediation guidance or a patch when appropriate. Public disclosure should wait until a fix or mitigation is available.

### Scope

In scope: authentication, authorization, data access, uploaded files, API endpoints, dependency vulnerabilities, and deployment configuration that may expose user or school data.

Out of scope: attacks requiring physical access, social engineering, denial-of-service testing against production systems, or issues caused by unsupported local modifications.

## 中文

### 支持版本

安全更新面向本仓库当前主分支提供。

### 报告漏洞

如果你发现安全问题，请私下联系项目维护者邮箱（可从github获取），不要直接创建公开 issue。报告中请包含受影响组件、复现步骤、预期影响，以及相关日志或截图。

### 响应流程

我们会尽快确认有效报告，调查问题，并在适当情况下提供修复方案或补丁。在修复或缓解措施可用之前，请暂缓公开披露。

### 范围

范围内：身份认证、权限控制、数据访问、上传文件、API 接口、依赖漏洞，以及可能暴露用户或学校数据的部署配置。

范围外：需要物理访问的攻击、社会工程、针对生产系统的拒绝服务测试，以及由未支持的本地修改导致的问题。
